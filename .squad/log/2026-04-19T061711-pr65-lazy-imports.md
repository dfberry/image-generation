## Summary

Refactored `generate.py` to defer `import torch`, `import diffusers`, and `from diffusers import DiffusionPipeline` from module-level to first real use. This eliminates the test-collection barrier identified in the codebase review synthesis — previously, `import diffusers` at module level blocked `pytest --collect-only` on machines without a full GPU stack installed.

## Changes (1 file, +80/−21 lines)

**`generate.py` only:**

1. **`_ensure_heavy_imports()` function** — Populates `torch`, `diffusers`, and `DiffusionPipeline` in `globals()` on first real use. Guarded by `"torch" not in globals()` check.

2. **PEP 562 `__getattr__`** — Module-level `__getattr__` exposes `torch`, `diffusers`, `DiffusionPipeline` for external code access (e.g. `unittest.mock.patch("generate.torch")`). Triggers lazy import on attribute access.

3. **`from __future__ import annotations`** — Defers evaluation of return-type annotations so `DiffusionPipeline` in type hints doesn't trigger an import at class/function definition time.

4. **8 call sites patched** — `_ensure_heavy_imports()` call added to: `get_device`, `get_dtype`, `_apply_performance_opts`, `load_base`, `load_refiner`, `apply_scheduler`, `generate`, `batch_generate`.

## Key Design Decision

Used `"torch" not in globals()` guard (not `is None` sentinel) so that `unittest.mock.patch("generate.torch")` sets the Mock in `globals()` before the guard runs, causing the real import to be skipped entirely. This preserves full backward compatibility with all 40+ existing `@patch` decorators — **zero test changes required**.
